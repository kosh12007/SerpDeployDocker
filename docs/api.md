## Модуль "Быстрая проверка позиций" (Старая система)

### POST `/run`

**Назначение:** Запускает процесс парсинга позиций.

**Параметры (form data):**
- `engine` — поисковая система (`google` или `yandex`)
- `queries` — список поисковых запросов, разделенных новой строкой
- `domain` — домен для отслеживания
- `loc_id_google` — ID локации Google (напр. 1000028 для Москвы)
- `loc_id_yandex` — ID региона Яндекса (напр. 213 для Москвы)
- `yandex_type` — тип парсинга Яндекса (`search_api` или `live_search`)
- `yandex_page_limit` — количество страниц для Live Search (по умолчанию 9)
- `google_page_limit` — количество страниц для Google (по умолчанию 10)

**Ответ:**
```json
{
  "status": "started",
  "session_id": "uuid-строка"
}
```

### GET `/status`

**Назначение:** Возвращает текущий статус процесса парсинга.

**Ответ:**
```json
{
  "status": "running|completed|error",
  "progress": "5/10",
  "current_query": "название запроса"
}
```

### GET `/download-session-results/<session_id>`

**Назначение:** Скачивает результаты сессии парсинга.

**Параметры (query string):**
- `format` — формат файла: `xlsx` (по умолчанию), `csv_utf8`, `csv_win1251`, `txt`

**Ответ:** Файл с результатами.

### DELETE `/delete-session/<session_id>`

**Назначение:** Удаляет сессию парсинга и связанные с ней данные.

**Ответ:**
```json
{
  "status": "success",
  "message": "Сессия удалена"
}
```

## Модуль "Проекты" (Новая система)

### GET `/projects`

**Назначение:** Отображает список проектов пользователя.

**Ответ:** HTML страница со списком проектов.

### POST `/projects/create`

**Назначение:** Создает новый проект.

**Параметры (form data):**
- `name` — название проекта
- `url` — URL сайта

**Ответ:** Редирект на страницу проекта.

### GET `/projects/<int:project_id>`

**Назначение:** Страница проекта с группами запросов и вариантами парсинга.

**Ответ:** HTML страница проекта.

### POST `/projects/<int:project_id>/update`

**Назначение:** Обновление данных проекта.

**Параметры (form data):**
- `name` — новое название
- `url` — новый URL
- `is_active` — статус активности

**Ответ:** Редирект обратно на страницу проекта.

### DELETE `/projects/<int:project_id>/delete`

**Назначение:** Удаление проекта и всех связанных данных.

**Ответ:**
```json
{
  "status": "success"
}
```

## Модуль "Выгрузка ТОП-10"

### GET `/top-sites`

**Назначение:** Страница модуля ТОП-10 сайтов.

**Ответ:** HTML страница с формой и историей задач.

### POST `/start-top-sites-parsing`

**Назначение:** Запускает задачу парсинга ТОП-10 сайтов.

**Параметры (form data):**
- `queries` — список запросов (по одному на строку)
- `search_engine` — поисковая система
- `loc_id` — локация/регион
- `device` — устройство
- `depth` — глубина выдачи (ТОП-10, ТОП-20 и т.д.)

**Ответ:**
```json
{
  "status": "success",
  "task_id": 123
}
```

### GET `/top-sites/status/<int:task_id>`

**Назначение:** Статус выполнения задачи.

**Ответ:**
```json
{
  "status": "running|completed|error",
  "progress": "45%"
}
```

### GET `/download-top-sites/<int:task_id>`

**Назначение:** Скачивает результаты задачи парсинга ТОП-10.

**Параметры (query string):**
- `format` — формат файла: `xlsx`, `csv_utf8`, `csv_win1251`, `txt`

**Ответ:** Файл с результатами (домены + полные данные).

### DELETE `/delete-top-sites/<int:task_id>`

**Назначение:** Удаляет задачу парсинга ТОП-10 и связанные с ней данные.

**Ответ:**
```json
{
  "status": "success"
```

## Общие API

### GET `/get-balance`

**Назначение:** Получает баланс из XmlRiver.

**Ответ:**
```json
{
  "balance": 12345.67
}
```

### GET `/api/locations`

**Назначение:** Возвращает список локаций Google для парсинга с пагинацией и поиском.

**Параметры (query string):**
- `search` — поисковый запрос (опционально)
- `page` — номер страницы (по умолчанию 1)

**Ответ (формат Select2):**
```json
{
  "results": [
    {"id": 1000028, "text": "Moscow,Moscow,Russia"},
    ...
  ],
  "pagination": {
    "more": true
  }
}
```

### GET `/api/yandex-regions`

**Назначение:** Возвращает список регионов Яндекса с пагинацией и поиском.

**Параметры (query string):**
- `search` — поисковый запрос (опционально)
- `page` — номер страницы (по умолчанию 1)

**Ответ (формат Select2):**
```json
{
  "results": [
    {"id": 213, "text": "Москва"},
    ...
  ],
  "pagination": {
    "more": true
  }
}
```

## Система настроек

### GET `/settings`

**Назначение:** Страница глобальных настроек (только для админов).

**Ответ:** HTML страница.

### POST `/settings/update`

**Назначение:** Обновление настроек приложения.

**Параметры (form data):**
- `MAX_QUERIES` — макс. количество запросов
- `ALLOW_REGISTRATION` — разрешить регистрацию
- `DEFAULT_USER_LIMITS` — лимиты по умолчанию

**Ответ:** Редирект на `/settings`.

## Аутентификация

### POST `/auth/login`

**Назначение:** Вход в систему.

**Параметры (form data):**
- `username` — имя пользователя или email
- `password` — пароль

**Ответ:** Редирект на главную или обратно на форму с ошибкой.

### POST `/auth/register`

**Назначение:** Регистрация нового пользователя.

**Параметры (form data):**
- `username` — имя пользователя
- `email` — email
- `password` — пароль

**Ответ:** Редирект на страницу входа.

### GET `/auth/logout`

**Назначение:** Выход из системы.

**Ответ:** Редирект на страницу входа.

### POST `/auth/forgot-password`

**Назначение:** Отправка email с токеном восстановления пароля.

**Параметры (form data):**
- `email` — email пользователя

**Ответ:** Сообщение об успехе/ошибке.

### POST `/auth/reset-password`

**Назначение:** Сброс пароля по токену.

**Параметры (form data):**
- `token` — токен из email
- `new_password` — новый пароль

**Ответ:** Редирект на страницу входа.
