# Исследование: Алгоритмы проверки уникальности текста

В данном документе описаны принципы работы сервисов проверки уникальности (Text.ru, Killer-Antiplagiat и др.) и план реализации собственного модуля.

---

## 1. Основные принципы работы

### Предварительная обработка (Cleaning)

- **Удаление стоп-слов:** Фильтрация предлогов, союзов и частиц.
- **Лемматизация:** Приведение слов к начальной форме (используя библиотеку `pymorphy2` или `NLTK`).
- **Канонизация:** Удаление пунктуации, приведение к нижнему регистру.

### Алгоритм шинглов (Shingles)

1. Текст разбивается на последовательности по `N` слов (обычно N=3..5).
2. Для каждой последовательности (шингла) вычисляется хэш (например, MD5 или CRC32).
3. Набор хэшей сравнивается с хэшами других текстов или ищется в поисковых системах.

---

## 2. Использование XMLRiver для проверки уникальности

**Вопрос:** Подойдет ли API XMLRiver для этой задачи?
**Ответ:** **Да, отлично подойдет.**

### Как использовать XMLRiver:

1. **Генерация запросов:** Разбить текст на самые «сильные» шинглы (фразы, несущие смысл).
2. **Поиск:** Отправлять каждый шингл как поисковый запрос в XMLRiver (Яндекс или Google).
3. **Анализ выдачи:**
   - Если XMLRiver возвращает результаты, смотрим на `snippet` (фрагмент текста на странице).
   - Сравниваем `snippet` с нашим исходным шинглом.
   - Если есть полное совпадение — фрагмент не уникален.
4. **Подсчет процента:** Уникальность = `(Количество уникальных шинглов / Общее количество проверенных шинглов) * 100%`.

## 4. Оптимизация и риски

- **Лимиты XMLRiver:** Проверка одного текста на 1000 знаков может потребовать 10-20 поисковых запросов. Нужно продумать кэширование результатов.
- **Скорость:** Поисковые запросы идут медленно. Нужно использовать асинхронность (`asyncio`) или очереди задач.
