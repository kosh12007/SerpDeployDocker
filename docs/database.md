## База данных

> **⚠️ Важная информация:** В проекте существуют две параллельные системы парсинга:
> - **Старая система:** основана на `parsing_sessions` (быстрая проверка)
> - **Новая система:** основана на `projects` (управление проектами)

### Система пользователей

#### Таблица `users`

**Назначение:** Хранение информации о пользователях системы.

**Поля:**
- `id` — уникальный идентификатор пользователя
- `username` — имя пользователя (уникальное)
- `email` — электронная почта (уникальная)
- `password_hash` — хеш пароля (Werkzeug)
- `limits` — количество доступных лимитов для парсинга
- `is_admin` — флаг администратора (BOOLEAN)
- `is_super_admin` — флаг суперадминистратора (BOOLEAN)
- `reset_token` — токен для восстановления пароля
- `reset_token_expires` — время истечения токена
- `created_at` — дата регистрации
- `last_login` — время последнего входа

#### Таблица `settings`

**Назначение:** Глобальные настройки приложения.

**Поля:**
- `setting_key` — ключ настройки (напр. `MAX_QUERIES`, `ALLOW_REGISTRATION`)
- `setting_value` — значение настройки
- `description` — описание настройки

---

### Новая система (Проекты)

#### Таблица `projects`

**Назначение:** Проекты парсинга с группами запросов.

**Поля:**
- `id` — идентификатор проекта
- `user_id` — владелец проекта
- `name` — название проекта
- `url` — URL сайта проекта
- `is_active` — активен ли проект (BOOLEAN)
- `created_at`, `updated_at` — временные метки

#### Таблица `query_groups`

**Назначение:** Группы запросов внутри проекта.

**Поля:**
- `id` — идентификатор группы
- `project_id` — связь с проектом
- `name` — название группы
- `created_at` — дата создания

#### Таблица `queries`

**Назначение:** Поисковые запросы.

**Поля:**
- `id` — идентификатор запроса
- `group_id` — связь с группой
- `text` — текст запроса
- `volume` — частотность (опционально)
- `created_at` — дата добавления

#### Таблица `parsing_variants`

**Назначение:** Варианты парсинга (комбинация SE + регион + устройство).

**Поля:**
- `id` — идентификатор варианта
- `project_id` — связь с проектом
- `name` — название варианта
- `search_engine_id` — поисковая система
- `search_type_id` — тип поиска
- `region_id` — регион (Yandex) или локация (Google)
- `device_id` — устройство (desktop/mobile)
- `created_at` — дата создания

#### Таблица `parsing_position_results`

**Назначение:** Результаты парсинга позиций.

**Поля:**
- `id` — идентификатор результата
- `variant_id` — вариант парсинга
- `query_id` — запрос
- `position` — позиция в выдаче (NULL если не найдено)
- `url` — найденный URL
- `top_10_urls` — JSON массив ТОП-10 URL (для кластеризации)
- `created_at` — дата проверки

---

### Старая система (Сессии)

#### Таблица `parsing_sessions`

**Назначение:** Сессии быстрой проверки позиций.

**Поля:**
- `id` — идентификатор записи
- `session_id` — UUID сессии
- `domain` — проверяемый домен
- `engine` — поисковая система (`google` или `yandex`)
- `status` — статус (`running`, `completed`, `error`)
- `user_id` — пользователь, запустивший проверку
- `created_at` — дата создания
- `spent_limits` — потраченные лимиты

#### Таблица `parsing_results`

**Назначение:** Результаты быстрой проверки.

**Поля:**
- `id` — идентификатор результата
- `query` — поисковый запрос
- `position` — позиция в выдаче
- `url` — найденный URL
- `processed` — статус обработки (`Yes`/`No`)

#### Таблица `session_results`

**Назначение:** Связь сессий и результатов (many-to-many).

**Поля:**
- `session_id` — идентификатор сессии
- `result_id` — идентификатор результата

### Справочники

#### Таблица `search_engines`

**Назначение:** Поисковые системы.

**Поля:**
- `id` — идентификатор
- `name` — название (`Google`, `Yandex`)
- `code` — код (`google`, `yandex`)

#### Таблица `search_types`

**Назначение:** Типы поиска.

**Поля:**
- `id` — идентификатор
- `name` — название (`Search API`, `Live Search`)
- `code` — код (`search_api`, `live_search`)

#### Таблица `devices`

**Назначение:** Типы устройств.

**Поля:**
- `id` — идентификатор
- `name` — название (`Desktop`, `Mobile`)
- `code` — код (`desktop`, `mobile`)

#### Таблица `yandex_regions`

**Назначение:** Регионы Яндекса для таргетинга.

**Поля:**
- `region_id` — идентификатор региона (напр. 213 = Москва)
- `region_name` — название региона

#### Таблица `locations`

**Назначение:** Локации Google для таргетинга.

**Поля:**
- `criteria_id` — уникальный идентификатор (напр. 1000028 = Москва)
- `name` — название
- `canonical_name` — каноническое название
- `parent_id` — родительская локация
- `country_code` — код страны
- `target_type` — тип таргетинга
- `status` — статус (`ENABLED`/`DISABLED`)
