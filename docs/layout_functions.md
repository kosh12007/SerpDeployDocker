# Документация по функциям в `layout.html`

В файле `templates/layout.html` определены несколько ключевых JavaScript-функций, которые отвечают за глобальный функционал приложения, такой как управление темами, отображение баланса и навигация.

---

## 1. `applyTheme()`

Эта функция отвечает за динамическое применение светлой или темной темы оформления на всем сайте.

### Логика работы:
1.  При вызове функция сначала ищет и удаляет старый тег `<link>` со стилями темы (с `id="theme-style"`), чтобы избежать конфликтов.
2.  Она проверяет значение `localStorage.theme`:
    *   Если `localStorage.theme` равен `'dark'`, применяется темная тема.
    *   Если в `localStorage` нет записи о теме, функция проверяет системные настройки пользователя через `window.matchMedia('(prefers-color-scheme: dark)')`. Если система предпочитает темную тему, она также применяется.
3.  В зависимости от условия, функция добавляет или удаляет класс `dark` у корневого элемента `<html>`.
4.  Создается новый тег `<link>` с соответствующим CSS-файлом (`theme-dark.css` или `theme-light.css`) и добавляется в `<head>` документа.
5.  Функция вызывается один раз при начальной загрузке страницы, а также при каждом клике на кнопку смены темы (`#theme-toggle`).

---

## 2. `fetchBalance()`

Эта функция асинхронно запрашивает и обновляет отображение баланса пользователя в шапке сайта.

### Логика работы:
1.  Функция вызывается при загрузке страницы (`DOMContentLoaded`) и при клике на кнопку "Обновить" (`#refresh-balance-btn`).
2.  Перед отправкой запроса она устанавливает текст в элементе `#balance-display` на "Загрузка...".
3.  Выполняется GET-запрос к эндпоинту `{{ url_for('main.get_balance') }}`.
4.  **При успешном ответе**:
    *   Полученный JSON-ответ парсится.
    *   Если в ответе есть поле `balance`, его значение отображается в `#balance-display`.
    *   В противном случае отображается "Ошибка".
5.  **При ошибке запроса** (сетевая проблема или ошибка сервера):
    *   В консоль выводится сообщение об ошибке.
    *   В `#balance-display` также отображается "Ошибка".

## 3. `redirectToProjectPositions()`

Эта функция обеспечивает "умное" перенаправление на страницу проверки позиций проекта.

### Логика работы:
1.  Функция привязана к пункту меню "Проверка позиций".
2.  Она пытается получить `selectedProjectId` из `localStorage`. Этот ID сохраняется, когда пользователь выбирает проект на странице списка проектов.
3.  **Если `selectedProjectId` найден**:
    *   Пользователь перенаправляется на страницу позиций для этого конкретного проекта (например, `/projects/123/positions`).
4.  **Если `selectedProjectId` не найден**:
    *   Пользователь перенаправляется на общую страницу со списком проектов (`/projects`), чтобы он мог выбрать нужный.
5.  Вся логика обернута в `try...catch` для обработки возможных ошибок при доступе к `localStorage`.
