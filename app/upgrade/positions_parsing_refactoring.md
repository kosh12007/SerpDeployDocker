# План рефакторинга модуля парсинга позиций

Этот документ описывает шаги по рефакторингу логики парсинга позиций проектов.

- [x] **Создание новой структуры папок:** Создать новую папку `app/positions_parsing`, а в ней — подпапки `core`, `db` и `utils`.
- [x] **Перенос логики парсинга:** Перенести функцию `run_parsing_in_thread` из `app/parsing.py` в новый файл `app/positions_parsing/core/parser.py`.
- [x] **Перенос работы с БД:** Создать файл `app/positions_parsing/db/operations.py` и перенести в него функции, связанные с созданием и обновлением сессий парсинга, а также списанием лимитов.
- [x] **Перенос утилит:** Создать файл `app/positions_parsing/utils/limits.py` и перенести в него логику расчета прогнозируемых лимитов.
- [x] **Обновление импортов:** Обновить все импорты в `app/blueprints/project_routes.py`, `app/blueprints/api_routes.py`, `app/blueprints/parsing_routes.py` и других файлах, чтобы они указывали на новые местоположения функций.
- [ ] **Тестирование:** Проверить, что после рефакторинга функционал парсинга позиций, включая отложенный запуск, подтверждение и списание лимитов, работает корректно.
