# Active Context: SERP Анализатор

## Текущее состояние

Проект находится в стадии активного развития и рефакторинга. Основной акцент делается на переходе от старой системы парсинга к новой — на базе Проектов и Вариантов парсинга.

## Текущие задачи

- **Рефакторинг**: Перенос логики из `main.py` в соответствующие Blueprints.
- **Унификация**: Совмещение старых результатов парсинга с новой структурой проектов.
- **Анализ проекта**: Проведение комплексного аудита кода и документации (текущая задача).
- **Проверка корректности удаления**: Гарантирование глубокого удаления всех связанных данных при удалении проекта.

## Недавние изменения

- Добавлена система модальных окон для улучшения UX.
- Реализованы новые Blueprints для AI и анализатора страниц.
- Улучшена система логирования во всех модулях.
- Внедрена глобальная CSRF-защита для всех форм и AJAX-запросов.
- Обновлена документация в папке `docs/` и Memory Bank.

## На что обратить внимание

- Модель `ParsingPositionResult` — центральная сущность для хранения позиций.
- Логика списания лимитов пользователей при запуске сессий парсинга.
- Каскадное удаление данных при удалении проектов (проверка таблиц `clusters`, `parsing_positions_sessions` и др.).
- Обязательное использование токена в `X-CSRFToken` для всех POST/PUT/DELETE запросов.
- Наличие `csrf_token` во всех HTML-формах.
